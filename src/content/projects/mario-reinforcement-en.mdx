## What it's about

A classic RL setup, built from scratch: an agent is supposed to learn the NES version of Super Mario Bros — purely from the pixels of the game screen, using Double Deep Q-Learning. The goal is a variant where you can watch the agent learn live.

## What's already in place

The codebase is modular (`agent.py`, `model.py`, `wrappers.py`, `train.py`, `config.py`):

- **Double DQN**: two networks (online and target), action selection decoupled from value bootstrapping — reduces the typical DQN overestimation of Q-values
- **CNN architecture**: 3 convolutional + 2 fully-connected layers, input is 4 stacked grayscale frames (84×84) so the agent can perceive velocity
- **Experience replay** and epsilon-greedy exploration
- **Reward shaping**: forward progress + coins + survival

## Status & next steps

The project is under active development — the architecture is in place, but a full training run hasn't happened yet. Up next:

1. First long training run with checkpoint persistence
2. Tuning hyperparameters (learning rate, epsilon decay, replay size)
3. Verifying the live training visualization
4. Documenting results — goal: clear World 1-1 reliably

## Why this project

You don't understand RL by reading about it — replay buffers, target networks and reward design are things you have to implement and debug yourself once. That's exactly what this lab is for.
